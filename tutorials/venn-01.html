We can remove the axes and the box by using axes=FALSE as an argument to the plot parameter. And then we should wrap the code in the function to create more easily all our next Venn diagrams.

sebsVenn = function (x,col=c("#cc888899","#8888cc99"),cex=1.6,...) {
    if (!is.data.frame(x) & !is.matrix(x)) {
        stop("Error: Not a two column matrix or data frame!")
    }
    if (ncol(x) != 2) {
        stop("Error: Not a two column matrix or data frame!")   
    }
    plot(c(1,2),c(1,1),xlim=c(0,3),ylim=c(0.5,1.7),
         pch=19,cex=50,axes=FALSE,
            xlab="",ylab="",
            col=col,...)
    x[,1][x[,1]==""]=NA
    x[,2][x[,2]==""]=NA    
    text(1,1.6,colnames(x)[1],cex=cex)
    text(2,1.6,colnames(x)[2],cex=cex)
    text(1.5,1,length(which(!is.na(intersect(x[,1],x[,2])))),cex=cex)
    text(0.5,1,length(which(!is.na(setdiff(x[,1],x[,2])))),cex=cex)
    text(2.5,1,length(which(!is.na(setdiff(x[,2],x[,1])))),cex=cex)
}   
sebsVenn(data.sam)

We can still change the plotting character size and give the diagram a title as all remaining arguments are delegated to the plot function.

Ok, so we can compute the values. We now modify our function above, so that it know differentiates between the data formats boolean and string and then computes separately the numbers and does finally the plot. Here this version:

delfsVenn = function (x,col=c("#cc888899","#8888cc99"),cex=1.6,...) {
    if (!is.data.frame(x) & !is.matrix(x)) {
        stop("Error: Not a two column matrix or data frame!")
    }
    if (ncol(x) != 2) {
        stop("Error: Not a two column matrix or data frame!")   
    }
    plot(c(1,2),c(1,1),xlim=c(0,3),ylim=c(-0.25,2.5),
         pch=19,cex=50,axes=FALSE,
            xlab="",ylab="",asp=1,
            col=col,...)
    # the changes
    if (class(x[,1]) == "logical") {
        is=length(which(x[,1] & x[,2]))
        ls=length(which(x[,1] & !x[,2]))
        rs=length(which(!x[,1] & x[,2]))
        os=length(which(!x[,1] & !x[,2]))
    } else {
        is=length(intersect(x[,1],x[,2]))
        ls=length(setdiff(x[,1],x[,2]))
        rs=length(setdiff(x[,2],x[,1]))
        os=""
    }   
    text(1,2.3,colnames(x)[1],cex=cex)
    text(2,2.3,colnames(x)[2],cex=cex)

    text(1.5,1,is,cex=cex)
    text(0.5,1,ls,cex=cex)
    text(2.5,1,rs,cex=cex)
    text(1.5,-0.2,os,cex=cex)
} 
delfsVenn(data.bool)
axis(1)
axis(2)

This function can now take any table which has one of the two different formats. It is generic for any data and also more flexible for the input data. It is even more reusable than the version before.

Fixing the aspect issue

Our code with the venn has one problem: if the figure space which is available is not square like our Venn diagram getâ€™s confused. Here an example with fig.width=5 and fig.height=5:

delfsVenn(data.bool)

The same effect could happend if the users uses different figure margins or changes the size of the figure in interactive session. Here the result with a figure size of width and height of 7 inches but setting a larger margin than the default:

par(mai=rep(1,4))
delfsVenn(data.bool)
axis(1)
axis(2)

We can fix this issue by manually setting the y-dimension to same before we start the plot and by setting the margins to the defaults and using the device width in inches for the calculation of the size of the plotting character for the circles. Further we calculate the circle size dynamically based on the available dimensions of the figure.

delfsVenn = function (x,col=c("#cc888899","#8888cc99"),cex=1.6,...) {
    if (!is.data.frame(x) & !is.matrix(x)) {
        stop("Error: Not a two column matrix or data frame!")
    }
    if (ncol(x) != 2) {
        stop("Error: Not a two column matrix or data frame!")   
    }
    # reset to useful values, slightly smaller than the defaults:
    # defaults: mai=c(1.02, 0.82, 0.82, 0.42)
    opar=par(mai=c(1, 0.8, 0.8, 0.4))
    # compute circle size
    circ.cex=50*par()$fin[1]/9
    plot(c(1,2),c(1,1),xlim=c(0,3),ylim=c(0.4,1.7),
         pch=19,cex=circ.cex,axes=FALSE,
            xlab="",ylab="",
            col=col,...)
    text(1,1.6,colnames(x)[1],cex=cex)
    text(2,1.6,colnames(x)[2],cex=cex)
    # the changes
    if (class(x[,1]) == "logical") {
        is=length(which(x[,1] & x[,2]))
        ls=length(which(x[,1] & !x[,2]))
        rs=length(which(!x[,1] & x[,2]))
        os=length(which(!x[,1] & !x[,2]))
    } else {
        is=length(intersect(x[,1],x[,2]))
        ls=length(setdiff(x[,1],x[,2]))
        rs=length(setdiff(x[,2],x[,1]))
        os=""
    }   
    text(1.5,1,is,cex=cex)
    text(0.5,1,ls,cex=cex)
    text(2.5,1,rs,cex=cex)
    text(1.5,0.4,os,cex=cex)
    par(opar)
}

Conclusion

Ok, our Venn diagram is now more robust in changing the figure size. We still have to keep this constant, but as long as we do this we are save. It would be as well possible to fix the issue with non-quadratic layouts, but that would be slightly too much burden.
